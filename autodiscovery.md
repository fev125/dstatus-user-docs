# 自动发现（批量安装 Agent）

## 入口

管理后台 → 自动发现（`/admin/autodiscovery`）

## 你能用它做什么

- 生成安装命令：把 Agent 装到被控机上
- 选择通信模式：被动（面板主动拉取）/ 主动上报（面板无法直连时使用）
- 审核/批准：如果你开启了审核，新节点需要你手动批准才能进入列表

## 推荐用法（最短路径）

1. 打开自动发现页面，选择“被动模式”（默认）
2. 如需修改通信端口再改（面板默认 `9999`）
3. 复制页面生成的安装命令，到被控机执行
4. 回到页面，看节点是否出现；如开启审核，点击“批准”

## 什么时候用主动上报

当面板无法访问被控机的通信端口时（例如被控机在 NAT/内网里），用主动上报。

注意：主动上报模式下，部分需要“面板直连 Agent”的功能会受影响（例如网络质量检测）。

## 常见问题

### 执行了安装命令，但节点没有出现

- 先等一会再刷新页面（新节点注册需要一点时间）
- 用“服务器管理”里的“安装探针/卸载探针”确认命令是否执行成功（见：/server-management）

### 节点出现了但显示离线

优先按 Agent 排障清单一步步查（见：/agent-guide）
