# 系统设置（管理后台）

## 入口
管理后台 → 管理设置：`/admin/setting`

## 这里能解决什么
- 改面板 Web 端口（默认 5555）
- 改管理员密码
- 改站点名称/网址
- 安装 Agent 下载源（安装脚本下载失败时用；页面里仍显示为 neko-status）
- 开关调试模式（日志会更详细）

补充：默认视图/桌面端视图/移动端视图、Logo、品牌色已迁移到“个性化设置 → 主题设置”。

## 最短操作

### 1）改端口（改完需要重启）
1. 修改“监听端口”，点击“保存设置”
2. 重启面板服务后，再用新端口访问

### 2）安装 Agent 下载失败（换源再重试）
1. 在“客户端下载地址配置”选择代理，或把“下载地址”换成你自己的镜像前缀
2. 回到服务器管理页，重新执行“初始化/安装 Agent”

## 常见问题

### 保存了端口，但访问还是旧端口
因为面板启动时只读取一次端口，所以改端口后必须重启面板服务。

### 无法登录时的服务器救援 {#忘记管理员密码}

这不是后台菜单，而是在**面板所在服务器**上执行的救援脚本。

常见触发场景：

- [面板迁移](./panel-migration.md) 或换机后，旧的动态登录入口一起被带过来，但通知收不到、也记不住真实路径
- [更新面板](./quick-start.md#更新面板) 后环境/域名变了，找不到登录入口
- 管理员密码忘记了，或恢复备份后密码与预期不一致

在安装目录所在机器执行（默认目录多为 `/opt/dstatus`；不是默认目录时，按脚本提示确认路径）：

```bash
curl -fsSL https://down.vps.mom/scripts/reset-password.sh -o reset-password.sh
bash reset-password.sh
```

脚本会先确认安装目录，再让你选：

1. 只重置管理员密码
2. 重置密码，并把登录入口恢复到 `/login`
3. 只把登录入口恢复到 `/login`

选完后用浏览器访问 `http(s)://你的面板地址:端口/login` 登录。登录成功后，立刻到本页把密码改成你自己的；若还要继续用动态入口，再到 [安全设置](./security-settings.md) 重新配置。

限制：

- 救援脚本不能解除已开启的 2FA；手机不可用时请用备份码
- 若已开启「禁用密码登录」，重置密码也进不去，需要先恢复可用的 GitHub 登录，或先用脚本恢复入口后再处理登录方式
