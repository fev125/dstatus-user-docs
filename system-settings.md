# 系统设置页面

## 页面路径
管理后台 → 系统设置（`/admin/setting`）

## 页面功能
- 配置系统监听端口
- 修改管理密码
- 配置站点基本信息
- 配置客户端下载地址

## 页面布局

### 账户设置
- **监听端口**：Web 界面监听的端口号（默认 5555）
- **管理密码**：管理员登录密码（修改时留空表示不修改）

### 站点设置
- **站点名称**：显示在页面标题和导航栏的系统名称
- **站点网址**：站点的完整访问地址（用于生成链接）
- **默认主题**：用户访问时的默认显示主题（卡片/列表）
- **站点 Logo URL**：用于邮件与部分 UI 的品牌展示
- **品牌主色**：用于邮件按钮等元素的品牌色（十六进制格式）

### 客户端设置
- **客户端下载地址配置**：配置 neko-status 客户端下载链接
  - 代理选择：选择代理加速服务（hub.gitmirror.com、gh-proxy.com、runn.i.ng）
  - 下载地址：客户端下载的基础链接前缀

## 操作步骤

### 修改监听端口
1. 进入"账户设置"区域
2. 修改"监听端口"字段
3. 点击页面底部的"保存更改"
4. **重要**：修改端口后需重启 DStatus 服务才能生效

### 修改管理密码
1. 进入"账户设置"区域
2. 在"管理密码"字段输入新密码
3. 留空表示保持当前密码不变
4. 点击"保存更改"
5. 密码修改后立即生效，下次登录需使用新密码

### 配置站点信息
1. 进入"站点设置"区域
2. 修改站点名称、站点网址等信息
3. 选择默认主题（卡片或列表）
4. 配置 Logo URL 和品牌主色（可选）
5. 点击"保存更改"
6. 配置立即生效，刷新页面查看效果

### 配置客户端下载地址
1. 进入"客户端设置"区域
2. 选择代理加速服务（可选）：
   - **不使用代理**：直接使用原始地址
   - **hub.gitmirror.com**：使用 GitMirror 代理
   - **gh-proxy.com**：使用 gh-proxy 代理
   - **runn.i.ng**：使用 runn.i.ng 代理
3. 输入客户端下载的基础链接前缀
4. 系统会自动根据代理选择转换下载链接
5. 点击"保存更改"

## 注意事项

- **端口修改**：修改监听端口后必须重启服务才能生效
- **密码安全**：密码使用 bcrypt 单向加密，无法显示原文
- **下载地址格式**：基础链接格式为 `https://example.com/files`，系统会自动添加架构后缀（如 `neko-status_linux_arm64`）
- **主题切换**：默认主题仅影响新用户首次访问，已登录用户可自行切换
