# 被控升级教程：v1.1 → v2.0.3

## 升级后你会得到什么

- 公网 IP 自动采集 -- Agent 自动检测并上报公网 IPv4/IPv6 地址
- TCP 连接数统计 -- 新增 `tcp_total`、`tcp_established`、`tcp_listen` 指标
- 稳定性改进 -- 修复多个边界情况和配置问题

主动模式还额外支持：

- Ping / TCPing 连通性检测
- MTR 路由追踪
- iperf3 带宽测试（需被控机已安装 iperf3 或支持自动安装）

## 选择升级方式

| 情况 | 做法 |
| --- | --- |
| 节点在线，编辑页能打开，已配置 SSH | [方法 A：编辑页 SSH 安装/升级](#方法-a-编辑页-ssh-安装-升级) |
| 节点离线 / 旧版残留多 / 没有 SSH | [方法 B：手动执行重装命令](#方法-b-手动执行重装命令) |
| 只能登录被控机器，面板不可用 | 直接在机器上重新执行安装命令 |

## 升级前确认

1. 确认主控面板版本 ≥ `v2.26.02.003`；低于此版本的主控可能不兼容新版被控，请先升级主控
2. 进入管理后台 → 设置中心 → 基础设置，确认"版本选择"已选到 `agent-v2.0.3`（或下载链接包含 `agent-v2.0.3`）；如果还是旧版本，先切换并保存

   <img src="/settings-agent-version.png" alt="设置中心 - 版本选择" style="max-width:720px;width:100%;" />
3. 被控机能访问 `down.vps.mom` 和 GitHub；国内主机访问不畅时，可在设置中心 → 代理加速中选择内置加速源，也支持自行填写加速前缀
4. 主动模式还需能访问面板地址

## 方法 A：编辑页 SSH 安装/升级

1. 管理后台 → 服务器管理 → 找到节点 → 编辑页
2. 有未保存改动先保存
3. 点击「SSH安装/升级探针」按钮，等待安装日志输出完成
4. 回列表确认节点重新上线、版本变为 `v2.0.3`

不顺利？不要反复点，直接走方法 B。

## 方法 B：手动执行重装命令

### Linux / macOS

1. 管理后台 → 服务器管理 → 在节点列表中找到该节点 → 操作菜单 → 复制"重装命令"
2. 到目标机器执行
3. 回面板确认上线

### Windows

1. 管理后台 → 服务器管理 → 找到节点 → 编辑页 → 下方「Windows 安装与重装」区域
2. 复制"Windows 重装命令"
3. 在目标机器 PowerShell 中执行
4. 重新登录一次 Windows 用户
5. 回面板确认上线

### OpenWrt

直接参考 [OpenWrt 运行教程](/Agent-OpenWrt运行教程)，按"重新部署当前版本"操作。

## 升级后确认

- [ ] 节点重新上线
- [ ] CPU / 内存 / 磁盘数据正常刷新
- [ ] 数据上报正常（主动 / 被动均应恢复）
- [ ] 版本确认：主动模式节点可在 服务器管理 → 主动探针状态 中查看版本列；被动模式节点暂无页面查看入口，可通过 SSH 登录被控机执行 `dstatus-agent --version` 确认

## 常见问题

**节点没回来？**
等 1-2 分钟 → 检查 `down.vps.mom` 可达性 → 主动模式检查面板地址 → 重新执行重装命令。

**还是旧版本？**
旧进程可能没退出，或执行了旧命令。重新从面板复制当前重装命令再执行一次。

**主动模式不上报？**
确认面板地址可达、节点仍为主动模式，重新执行面板生成的主动模式命令。

**Windows 没有自动启动？**
重新登录当前用户，如仍不行，重新执行 Windows 重装命令。
