# 被控升级教程：v1.1 升级到 v2.0.3

这篇教程只讲一件事：你手上的旧版被控（Agent）如果还是 `v1.1`，现在怎么安全升级到 `v2.0.3`。

本文只面向用户操作，不讲内部实现。

## 先判断你该走哪条路径

先看你现在的节点状态：

| 你现在的情况 | 推荐做法 |
| --- | --- |
| 节点还在面板里，当前在线，编辑页能正常打开 | 优先用面板里的“升级探针” |
| 节点还在面板里，但已经离线很久，或者你准备顺手把旧配置一起收口 | 优先用“重装命令” |
| 你现在只能登录被控机器，面板里的升级入口用不了 | 直接重新执行当前安装/重装命令 |
| Windows / OpenWrt 老版本，或者你不确定旧版残留了什么 | 不要手工替换零散文件，优先按当前文档重装 |

如果你拿不准，直接按这一条执行：

- `v1.1 -> v2.0.3` 这种跨度较大的升级，默认优先“重新执行当前面板生成的安装/重装命令”，不要优先选手工替换单个二进制。

## 升级前先确认

开始前先做 4 件事：

1. 记一下这台节点当前是“被动模式”还是“主动上报”
2. 确认被控机还能访问 `down.vps.mom`
3. 如果你是主动模式，确认被控机还能访问你的面板地址
4. 如果旧版机器里还能看到旧名字、旧目录或旧服务名，不用先手动清理，按当前教程升级即可

补一句：

- 升级过程中节点短暂离线、重连一次，通常是正常现象。

## 路径 A：通过面板升级

适合：

- 节点还在面板里
- 当前在线或基本可管理
- 你希望直接从页面完成升级

步骤：

1. 打开管理后台
2. 进入“服务器管理”
3. 找到这台节点，打开编辑页
4. 如果页面提示有未保存改动，先保存
5. 点击“升级探针”
6. 等待页面提示完成，再回到列表或详情页确认节点重新上线

升级后你应该看到：

- 节点短暂重连后恢复在线
- 数据继续刷新
- 如果页面有版本显示，版本应更新到 `v2.0.3`

如果这条路不顺：

- 不要反复点升级
- 直接切到下面的“路径 B：重新执行当前安装/重装命令”

## 路径 B：重新执行当前安装/重装命令

这条路径更适合：

- 老版本跨度大
- 节点离线
- 你怀疑旧版残留较多
- 你希望把当前节点配置一起重新装回去

### Linux / macOS

推荐顺序：

1. 回到面板里的节点编辑页，复制当前生成的安装命令或重装命令
2. 到目标机器重新执行
3. 执行完成后，回面板看节点是否重新上线

建议：

- 如果你是从很早的旧版升上来，优先选“重装命令”
- 不要自己手工找旧目录里替换零散文件

### Windows

推荐顺序：

1. 回到面板里的节点编辑页
2. 复制当前的“Windows 安装命令”或“Windows 重装命令”
3. 在目标 Windows 主机的 PowerShell 里执行
4. 执行完成后重新登录一次系统，确认探针自动拉起
5. 回面板确认节点恢复在线

建议：

- `v1.1 -> v2.0.3` 默认优先用当前页面生成的 Windows 命令
- 不要只手工覆盖旧 exe 后就结束；旧版残留的启动方式和目录口径可能不一致

### OpenWrt

OpenWrt 建议不要在这篇教程里照着通用步骤硬套。

直接看：

- [OpenWrt 运行教程](/Agent-OpenWrt运行教程)

如果你是从 `v1.1` 升上来，默认按“重新部署当前版本”处理更稳，不建议只手工替换单个旧文件。

## 升级后怎么确认成功

至少确认下面 4 件事：

1. 节点重新上线
2. CPU / 内存 / 磁盘等数据继续刷新
3. 如果页面有版本显示，已变成 `v2.0.3`
4. 模式符合预期：
   - 被动模式：面板还能正常拿到数据
   - 主动模式：节点还能继续向面板上报

## 常见问题

### 1）升级后节点没回来

先按这个顺序查：

1. 先等 1-2 分钟再刷新
2. 确认目标机器能访问 `down.vps.mom`
3. 如果是主动模式，确认目标机器还能访问面板地址
4. 重新执行一次当前页面生成的重装命令

### 2）还是显示旧版本

常见原因：

- 旧进程没有真正退出
- 你执行的是旧命令
- 只替换了文件，但没有按当前方式重新拉起

处理：

1. 回到面板重新复制一次当前命令
2. 优先执行“重装命令”而不是继续手工覆盖
3. 再回页面确认版本

### 3）主动模式更新后不上报

先查：

1. 面板地址是否还能访问
2. 当前节点是否还是主动模式
3. 重新执行当前页面生成的主动模式命令

如果你是从很旧的版本升上来，这类问题通常用“重装命令”比“反复点升级”更稳。

### 4）Windows 更新后没有自动启动

先做两件事：

1. 重新登录一次 Windows 当前用户
2. 再回面板确认节点是否恢复

如果还不行，直接重新执行当前“Windows 重装命令”。
