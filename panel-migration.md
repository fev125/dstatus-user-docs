# 面板迁移教程

本文用于把旧机器上的 DStatus 面板迁移到新机器，并尽量保留你的配置与数据。

## 迁移前准备

- 确保你能登录旧面板和新面板的管理员账号。
- 部署入口在 `https://client.vps.mom`，安装脚本会从 `down.vps.mom` 拉取资源（新机器需要能访问该域名）。
- 如果开启过 2FA、GitHub 登录或禁用密码登录，迁移前请确认至少有一种可用的后台登录方式；使用 GitHub 登录且迁移后域名变化时，需要同步更新 GitHub OAuth 回调地址。

## 方案 A（推荐）：面板内备份 + 恢复

适合大多数用户，操作简单，出错概率低。

### A1. 在旧面板导出备份

1. 登录旧面板后台。
2. 打开「高级设置」页面：`/admin/advanced-settings`。
3. 在「数据库管理」卡片点击「下载压缩备份（gzip）」并保存到本地。

如果你只想迁移“配置 + 节点列表”，不需要迁移历史监控数据，可以下载「轻量备份」。

### A2. 在新面板恢复备份

1. 在新机器按 `client.vps.mom` 的部署流程安装并启动面板。
2. 登录新面板后台 → 打开 `/admin/advanced-settings`。
3. 在「数据库管理」卡片点击「恢复数据库」，上传你在旧面板下载的备份文件。
4. 等待恢复完成（恢复过程中页面可能会短暂断开或需要刷新）。
5. 刷新页面后重新登录，检查节点列表/分组/通知等是否正常。

## 方案 B（备选）：只拷贝数据目录

适合你无法在面板内上传备份文件，或者更习惯用 SSH 操作。

说明：默认情况下，面板的核心数据与配置都在 `data` 目录内（例如数据库文件），迁移 `data` 即可完成主要迁移。

### B1. 停止旧面板

在旧机器执行其一即可：

- `systemctl stop dstatus`
- 或进入安装目录后执行：`docker-compose down`

### B2. 拷贝 data 目录到新机器

默认安装目录是 `/opt/dstatus`，需要迁移的目录是：

- 旧机器：`/opt/dstatus/data`
- 新机器：`/opt/dstatus/data`

请用你熟悉的方式，把旧机器的 `data` 目录内容完整复制到新机器对应目录。

如果你的面板不在默认安装目录，请迁移实际安装目录下的 `data` 目录。

### B3. 启动新面板并验证

在新机器执行其一即可：

- `systemctl start dstatus`
- 或进入安装目录后执行：`docker-compose up -d`

打开新面板，确认节点列表、分组、通知等数据已迁移成功。

## 常见问题

### 1) 迁移后需要重新登录吗？

通常需要。恢复数据库后，登录信息会以你备份时的状态为准。

### 2) 迁移后地址/端口变了怎么办？

如果你对外访问使用域名或反向代理，需要把域名解析/代理配置指向新机器的面板地址与端口。

### 3) 迁移后 SSH 连接失败怎么办？

先确认目标服务器 IP、端口、用户名和防火墙没有变化。

如果只有少数节点无法连接，可以进入服务器编辑页重新保存 SSH 认证；如果你不想把 SSH 认证保存在面板里，也可以改用「本地 SSH 保险箱」。

### 4) 迁移后找不到登录入口，或密码对不上怎么办？

因为动态登录入口和密码都在数据库里，所以恢复备份 / 拷贝 `data` 之后，新机器会沿用旧面板的入口和密码。迁移后域名、通知渠道或记忆对不上时，就容易出现「入口不见了」或「密码不对」。

按顺序处理：

1. 先试固定提示页 `/admin-login`（开启过动态入口时，可能还能通过通知找回真实入口）
2. 仍进不去时，到新机器上跑服务器救援脚本，按提示选择「恢复登录入口到 `/login`」和/或「重置管理员密码」：见 [无法登录时的服务器救援](./system-settings.md#忘记管理员密码)
3. 登录成功后，立刻改成你自己的密码；需要继续用动态入口时，再到 [安全设置](./security-settings.md) 重新配置

如果是 2FA 设备不可用，请使用备份码。救援脚本不能解除已有的 2FA 验证。
