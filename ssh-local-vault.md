# 本地 SSH 保险箱

## 这是什么

本地 SSH 保险箱就是把你的密码或私钥只放在当前浏览器里。

这样做有两个好处。

- 你不用每次都重新输入
- 你也不用把密码长期存到面板里

## 能用在哪些地方

本地 SSH 保险箱现在支持这些操作。

- 测试连接
- SSH 安装更新
- 终端连接
- 终端里直接执行命令
- 终端里的 Agent

只要当前节点已经在这个浏览器里存过认证，而且保险箱已经解锁，这些功能就会优先使用本地认证。

## 怎么使用

### 方式一

如果你正在编辑节点，并且已经手动填了密码或私钥，可以直接点 `保存到本地`。

适合这些情况。

- 你不想每次都重填
- 你不想把密码长期留在面板里

### 方式二

如果这个节点的密码或私钥已经保存在面板里，可以点 `迁移到本地`。

这样会先把认证保存到当前浏览器，再清理面板里原来保存的认证。

## 你看到的按钮是什么意思

### 当前节点

- `保存到本地`
  把你当前正在输入的密码或私钥保存到这个浏览器
- `迁移到本地`
  把面板里已经保存的密码或私钥搬到这个浏览器

### 本地备份

- `导出`
  导出这个浏览器里已经保存的全部本地数据
- `导入`
  用导出的文件恢复全部本地数据
- `锁定`
  临时锁住保险箱
- `清空本地`
  删除这个浏览器里全部本地数据

## 数据怎么走

```text
你发起操作

    |

    |     |
    |     v
    |   直接使用
    |
    +-- 没有

          v
    再看这个浏览器里有没有本地认证
          |

          |     |
          |     v

          |

          |     |
          |     v
          |   再看面板里有没有已保存认证
          |

```

## 终端里的规则

终端页现在也会优先使用本地 SSH 保险箱。

你在服务器管理页已经保存过本地认证后，到了终端页，系统会自动尝试使用这一份本地认证。

所以你通常不用再在终端里重新填一次。

如果终端里点了 `清除临时配置`，只会清掉这次终端连接用到的临时信息。

它不会清掉你浏览器里的本地 SSH 保险箱。

## 注意事项

- 本地 SSH 保险箱只属于当前浏览器
- 换浏览器或换电脑后不会自动跟过去
- 需要提前导出，再在新环境导入
- 保险箱口令忘记后无法找回
- 导出时导出的是这个浏览器里的全部本地数据
- 导入时会替换这个浏览器里原有的本地数据
- 锁定后需要重新输入口令才能继续使用

## 什么时候适合用

推荐这样用。

- 你想用 SSH 安装更新
- 你经常要测试连接
- 你经常会打开终端
- 你不想把密码长期放在面板里

## 常见问题

### 为什么我换了浏览器就不能用了

因为本地 SSH 保险箱只保存在你原来的那个浏览器里。

### 为什么导入后还是要重新输入口令

因为导入只恢复数据，不会自动解锁。

### 为什么终端还能连上

因为终端现在也会优先读取当前浏览器里的本地认证。
