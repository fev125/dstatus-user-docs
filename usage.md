# 使用指南

装完面板后，下一步是把服务器接进来并确认数据正常。

## 添加服务器（两种方式）

### 方式 A：自动发现（适合批量）

1. 管理后台 → 自动发现（`/admin/autodiscovery`）
2. 复制页面生成的安装命令，在目标服务器上执行
3. 回到页面刷新，看到节点出现即成功；开了审核的需点"批准"

### 方式 B：手动添加（适合 1-2 台）

1. 管理后台 → 服务器管理（`/admin/servers`）→ 新增服务器
2. 填写 SSH 连接信息并保存
3. 按提示安装 Agent

## 模式怎么选

两种模式：**被动模式（默认）**、**主动上报**。

| 情况 | 模式 | 要求 |
| --- | --- | --- |
| 面板能直连服务器 | 被动模式 | 面板 → 服务器通信端口可达（默认 `9999`） |
| 服务器在内网 / 没公网 | 主动上报 | 服务器 → 面板地址可达（默认 `5555`） |
| 两边都不通 | 先解决网络 | 端口放行 / 转发 / 中继，再按上面选模式 |

常见情况：

- **没有公网 IP / 不能开入站端口**：用主动上报，服务器能访问面板地址即可。
- **只允许少数出站端口**：面板端口要在允许范围内，主动上报才能连上。
- **只有 IPv6**：被动和主动都一样，关键是双方能互相访问到对方。
- **需要中继/穿透**：DStatus 不会自动打洞，先用自己的方式把网络打通，再选模式。

> 安装命令会从 `down.vps.mom` 下载资源，下载失败会导致节点不出现。

## 常见问题

### 节点没出现

安装命令没执行成功。

- 重新复制安装命令执行一次
- 确认服务器能访问 `down.vps.mom`

### 节点出现了但离线 / 数据不更新

面板拿不到数据。

- 被动模式：确认通信端口已放行（默认 `9999`）
- 主动上报：确认服务器能访问面板地址（默认 `5555`），检查上报配置

更完整排查见：[Agent 使用指南](/agent-guide)

### 面板打不开

端口没放行或被占用。

- 确认访问的端口正确（默认 `5555`）
- 确认安全组 / 防火墙已放行该端口
