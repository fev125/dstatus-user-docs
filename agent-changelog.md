## agent-v2.2.4

- 手动诊断 MTR 在超时后仍会返回已测得的部分结果，不再长时间无响应
- 探针升级可走面板配置的下载加速前缀/镜像，失败后回退官方源
- 上报中补充默认路由网卡信息，便于正确识别主网卡
- 若干安全与兼容问题修复

## agent-v2.2.3

- 修复切换主动/被动上报后配置未真正生效的问题（相关变更会触发探针重启以加载新配置）

## agent-v2.2.2

- 主动上报模式下探针不再监听本地 API 端口

## agent-v2.1.1

- 网络与磁盘速率的采样窗口调整为 1 秒

## agent-v2.1.0

- 扩展连接与系统诊断信息采集
- 支持 GPU 相关过滤
- 完善 Windows 诊断能力

## agent-v2.0.5

- 面板下发的托管配置可持久化，重启后仍保留
- 支持切换节点上报地址
- 本地 HTTP 接口在密钥无效或未配置时拒绝启动对外服务
- 本地判定已过期的任务仍会执行并回传，减轻时钟偏差导致的漏报

## agent-v2.0.4

- 被动节点可回填探针版本号
- 优化 Ping 采样
- 提升轻量诊断任务的并发处理能力
- 支持通过安装脚本在 Windows 上部署探针

## agent-v2.0.3

- 与上一版行为一致

## agent-v2.0.2

- 增加公网 IP 采集与上报
- 主动上报间隔下限调整为 1 秒
- 若干安全与兼容问题修复

## agent-v2.0.1

- 手动诊断支持 MTR 与 iperf3（含缺失时自动安装、镜像兜底，以及更可读的失败提示）
- 新增 inode、磁盘 IO、连接数等采集指标
- 支持主控向单节点下发探针配置
- 主动探针可展示版本并支持手动升级
- 完善任务轮询与回传
- 上报信息补齐主机名

## agent-v1.1.9

- 被控正式版起点；后续能力见更高版本
