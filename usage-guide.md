# DStatus 操作指南

面向普通用户的完整操作手册，涵盖日常监控、配置调整和故障排除。

## 📊 仪表板使用

### 主要界面说明
- **服务器列表**：显示所有监控服务器的实时状态
- **总体统计**：顶部显示在线/离线服务器数量和网络流量概览
- **性能指标**：每个服务器卡片显示CPU、内存、磁盘、网络等关键指标
- **状态指示灯**：绿色=在线，红色=离线

### 快速导航
- **刷新按钮**：手动刷新所有监控数据
- **筛选按钮**：按状态、分组、到期时间筛选服务器
- **排序选项**：点击列标题可按CPU、延迟等指标排序

## 🔧 服务器管理

### 添加服务器
#### 自动发现方式
1. 登录管理后台 → 点击"自动发现"
2. 生成注册密钥（可选设置需要审核）
3. 在目标服务器执行安装命令（命令在页面显示）
4. 等待服务器自动注册（需网络连通）

#### 手动添加方式
1. 管理后台 → "服务器管理" → "新增服务器"
2. 填写SSH信息：
   - 服务器名称
   - IP地址或域名
   - SSH端口（默认22）
   - 用户名和密码
3. 点击"测试连接"验证
4. 保存后点击"安装探针"部署监控程序

### 服务器操作
- **编辑**：点击服务器卡片右侧编辑按钮，修改配置
- **卸载**：编辑页面点击"卸载命令"，复制执行后可删除服务器记录
- **分组**：将服务器拖拽到不同分组，便于管理

## ⚙️ 配置设置

### 基本设置
访问管理后台 → "系统设置"：
- **站点名称**：修改系统显示名称
- **监听端口**：修改Web界面访问端口（需重启服务）
- **主题设置**：选择深色/浅色主题
- **语言**：切换界面语言

### 监控配置
- **更新间隔**：设置数据刷新频率（默认5秒）
- **监控项目**：选择监控CPU、内存、网络等指标
- **告警阈值**：设置CPU>80%、内存>90%时触发告警

### 通知配置
- **Telegram通知**：
  1. 获取Bot Token（@BotFather）
  2. 获取Chat ID
  3. 在"通知设置"页面填写配置
  4. 点击"测试通知"验证
- **邮件通知**：配置SMTP服务器信息

### 个性化设置
- **壁纸设置**：上传自定义背景图片，调整亮度和模糊效果
- **毛玻璃效果**：启用/禁用界面模糊效果
- **数据保留**：设置历史数据保存天数（默认90天）

## 🚨 故障排除

### 常见问题解决

#### 1. 服务器显示离线
**可能原因**：
- 网络连通性问题
- 探针程序异常
- SSH连接失败

**解决步骤**：
1. 检查服务器网络是否能访问监控面板
2. 登录服务器执行`ps aux | grep dstatus`检查探针进程
3. 如果进程不存在，重新执行安装命令
4. 检查服务器防火墙是否阻止了通讯端口（默认19999）

#### 2. 数据更新不及时
**可能原因**：
- WebSocket连接问题
- 浏览器缓存问题

**解决步骤**：
1. 点击刷新按钮手动更新
2. 按Ctrl+F5强制刷新页面
3. 检查浏览器控制台是否有WebSocket错误
4. 尝试更换浏览器或清除缓存

#### 3. 无法添加服务器
**可能原因**：
- SSH配置错误
- 权限不足
- 网络问题

**解决步骤**：
1. 确认SSH用户名和密码正确
2. 确保SSH端口正确（默认22）
3. 测试SSH连接：`ssh username@ip -p port`
4. 检查服务器防火墙是否允许SSH连接

#### 4. 通知不发送
**可能原因**：
- 配置错误
- 网络问题
- Token过期

**解决步骤**：
1. 重新获取Telegram Bot Token
2. 验证Chat ID正确
3. 检查管理后台"测试通知"功能
4. 确保服务器能访问Telegram API

#### 5. 界面加载缓慢
**可能原因**：
- 服务器性能不足
- 网络延迟高
- 浏览器扩展冲突

**解决步骤**：
1. 关闭不必要的浏览器扩展
2. 清除浏览器缓存和Cookie
3. 尝试在无痕模式下访问
4. 检查服务器CPU和内存使用情况



### 安全注意事项
- **修改默认密码**：首次登录后立即修改admin密码
- **启用两步验证**：在安全设置中启用2FA
- **定期备份**：定期备份数据库和配置文件
- **更新系统**：及时更新DStatus到最新版本
